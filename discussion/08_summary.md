# 8. 토론 최종 요약

> 4명의 팀원(PM, 프론트엔드, 경제 도메인 전문가, 데이터 아키텍트)이 7개 주제에 걸쳐 심도 있는 토론을 진행한 결과입니다.

---

## 프로젝트 비전

**"경제 지표 간의 인과관계를 직관적으로 시각화하여, 투자 초보자가 하나의 경제 이벤트가 자신의 투자에 미치는 연쇄 영향을 즉시 파악할 수 있게 하는 인터랙티브 관계도"**

### 핵심 가치
1. **직관성** — 터치 한 번으로 연쇄 관계 파악
2. **정확성** — 단순 공식이 아닌, 조건과 맥락이 포함된 관계
3. **성장성** — 사용자가 학습하면서 관계도가 함께 성장

### 킬 피처
- **"이 지표가 오르면?/내리면?" 시뮬레이션** — 노드 탭 → 전체 연쇄 영향을 화살표로 시각화
- 이것이 이 앱을 단순한 정적 다이어그램이 아닌 **인터랙티브 학습 도구**로 만드는 핵심

## 데이터 구조

### MVP: `data/data.json` 단일 파일
```json
{
  "version": "1.0",
  "lastUpdated": "2026-04-03",
  "categories": { ... },
  "nodes": [
    {
      "id": "rate",
      "label": "기준금리",
      "sub": "Fed Funds Rate",
      "category": "정책",
      "description": "...",
      "interpretation": { "up": "...", "down": "..." },
      "schedule": "FOMC 연 8회",
      "dataSource": { "fred": "FEDFUNDS" },
      "tags": ["핵심"]
    }
  ],
  "links": [
    {
      "id": "rate-bond",
      "source": "rate",
      "target": "bond",
      "type": "반비례",
      "strength": 5,
      "timeframe": "즉각",
      "description": "금리↑ → 채권가격↓",
      "detail": "...",
      "conditions": "...",
      "exceptions": "..."
    }
  ]
}
```

### 설계 원칙
- 필수 필드 최소화 (id, label, category, description만 필수)
- 선택 필드로 점진적 풍부화
- 데이터와 뷰(레이아웃) 분리
- 하위 호환성 유지

### 향후 확장
- Phase 2: `events.json` (이벤트 기록), `scenarios.json` (시나리오)
- Phase 3: 사용자 커스텀 데이터 (localStorage)

## MVP 기능 (Phase 1)

| 기능 | 설명 |
|------|------|
| 외부 데이터 로딩 | data.json에서 노드/링크 로드 |
| Force-directed 레이아웃 | D3.js 자동 배치 + 드래그 조정 |
| 노드 탭 → 하이라이트 | 직접 연결 노드/링크 강조 |
| 바텀시트 팝업 | 지표 상세 정보 (스와이프 지원) |
| **연쇄 효과** | **BFS 기반 다단계 파급 효과 시각화** |
| **시뮬레이션** | **"오르면?/내리면?" → 전체 영향 방향 표시** |
| 카테고리 필터 | 특정 카테고리 보기/숨기기 |
| 검색 | 지표명으로 검색 → 포커스 |
| 핀치줌/패닝 | 관계도 확대/축소/이동 |

### 주요 논쟁과 결론
- **시뮬레이션을 MVP에 포함할지**: 원래 Phase 2였으나, 연쇄 효과의 자연스러운 확장이므로 MVP로 격상 (전원 합의)
- **3단계 이상 연쇄의 정확성**: "참고용" 경고 표시로 해결 (ECO 주장 반영)
- **부호 전파(비례x반비례=반비례)**: 수학적으로 구현, 조건부 관계는 "?" 표시

## UI/UX 설계

### 화면 구조
```
[검색바] [필터] [메뉴]     ← 상단

   관계도 (SVG, D3.js)     ← 중앙 (핀치줌/패닝)

[바텀시트: 지표 정보]      ← 하단 (스와이프)
[오르면?] [내리면?]         ← 시뮬레이션 버튼
```

### 인터랙션
- 탭 → 1차 연결 하이라이트 + 바텀시트
- 바텀시트 "오르면?/내리면?" → 시뮬레이션 모드
- 빈 곳 탭 → 선택 해제
- 노드 드래그 → 위치 조정
- 핀치줌/패닝 → 관계도 탐색

### 기술 선택
- **D3.js v7** (CDN) — force layout, zoom, drag
- 다크 테마 유지 (가독성 개선)
- 노드 터치 영역 최소 44px

## 기술 스택 (최종 합의)

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | 없음 (Vanilla JS) | 코딩 초보 유지보수 가능 |
| 시각화 | D3.js v7 (CDN) | force layout 필수, 풍부한 레퍼런스 |
| 빌드 도구 | 없음 | GitHub Pages 직접 배포 |
| 스타일 | CSS (단일 파일) | 단순함 유지 |
| 데이터 | JSON 파일 | 서버 불필요 |
| 사용자 저장 | localStorage | 서버 불필요 |
| 배포 | GitHub Pages | 현재 인프라 유지 |
| AI 연동 | Claude API (Phase 2) | 사용자 API 키 방식 |

## 핵심 논쟁 요약

| 논쟁 | 결론 | 근거 |
|------|------|------|
| 시뮬레이션을 MVP에? | **포함** | 연쇄효과의 자연스러운 확장, 킬 피처 |
| JS 파일 분리 구조? | **MVP에서는 단일 HTML + 외부 JSON** | 코딩 초보 접근성, 실용주의 |

| 노드 위치 관리? | **force-directed 자동 + localStorage 저장** | 확장성, 사용자 커스터마이징 |
