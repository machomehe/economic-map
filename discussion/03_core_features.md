### PM: 기능 브레인스토밍부터 하고, 우선순위를 정합시다

제가 사용자 시나리오 기반으로 정리한 기능 후보 목록입니다:

**A. 관계도 탐색 (핵심)**

3. **경로 탐색** — "A가 B에 영향을 미치는 경로" 표시
4. 카테고리별 필터/그룹핑
5. 검색 기능 — 지표 이름으로 검색

6. 노드 상세 정보 (설명, 해석 가이드, 발표 일정)
7. 관계 상세 정보 (왜 이렇게 되는지 설명)
8. **시나리오 시뮬레이션** — "금리가 오르면?" 시뮬레이션 모드
9. 경제 용어 사전 (팝업 내 링크)

**C. 개인화**

12. 즐겨찾기/북마크 시나리오
13. 학습 이력 추적

**D. 자동화**
14. 톡방 캡처 → AI 분석 → 데이터 제안
15. economic-dashboard 데이터 연동 (실시간 값 표시)
16. 이벤트 알림 (FOMC 일정 등)

**E. 공유/내보내기**
17. 관계도 스크린샷 공유
18. 시나리오 공유 (URL)

### ECO: 2번 "연쇄 효과"가 가장 중요합니다, 강력히 주장합니다

이유: 투자에서 가장 가치 있는 건 **"1차 효과가 아니라 2차, 3차 효과를 아는 것"**이거든요.

예를 들어:
- 초보: "금리 올랐네" → 끝
- 중급: "금리 올랐으니 채권 떨어지겠네" → 1차 효과
- 고급: "금리↑ → 달러↑ → 원화↓ → 외국인 매도 → 내 삼성전자 떨어지겠다" → 연쇄 효과

이 프로젝트의 존재 이유가 **초보를 중급-고급으로 올리는 것**이라면, 연쇄 효과 시각화가 킬 피처입니다.

### FE: 연쇄 효과 구현 방법 제안

기술적으로 두 가지 접근:

**접근 1: 단계별 하이라이트 (애니메이션)**
- 노드 터치 → 1차 연결 하이라이트 (0.3초 후) → 2차 연결 하이라이트 (0.6초 후) → 3차...
- 각 단계가 색이 점점 연해지거나, 파동(ripple) 효과
- 장점: 직관적, 시간 순서 표현 가능
- 단점: 너무 많은 노드가 연결되면 거의 다 하이라이트됨

**접근 2: 인터랙티브 확장**
- 노드 터치 → 1차 연결만 하이라이트
- 연결된 노드를 다시 터치 → 그 노드의 1차 연결 추가 하이라이트
- 사용자가 스스로 경로를 탐색
- 장점: 사용자 주도 학습, 정보 과부하 방지
- 단점: 여러 번 터치해야 함

**추천: 접근 1 + 2 결합**
- 기본: 접근 1 (자동 단계별 하이라이트)
- 하이라이트 깊이(depth) 조절 슬라이더: 1단계/2단계/3단계
- 특정 경로를 터치로 고정(pin) 가능

### PM: 반론 — 깊이 슬라이더는 모바일에서 어색할 수 있어요

슬라이더 대신 **탭/더블탭으로 깊이 전환**하는 건 어떨까요?
- 한 번 탭: 1차 연결
- 더블 탭 또는 길게 누르기: 전체 연쇄 효과
- 이게 모바일 인터랙션에 더 자연스럽습니다

### FE: 좋은 의견입니다. 다만 더블탭은 줌과 충돌할 수 있어요

모바일에서 더블탭이 화면 줌으로 인식될 수 있습니다. 대안:
- 한 번 탭: 1차 연결 + 팝업
- 팝업 안에 **"연쇄 효과 보기" 버튼** → 클릭하면 전체 체인 표시
- 이러면 의도적 액션이라 실수 방지도 됨

### DA: 연쇄 효과를 위한 데이터 구조 확인

현재 links.json의 source/target 관계로 그래프를 구성하면, BFS/DFS로 N-depth 탐색이 가능합니다. 추가로 필요한 건:

1. **방향성**: 현재 "비례/반비례"만 있는데, 인과의 방향도 중요. "금리→채권" 관계에서 금리가 원인, 채권이 결과. 이미 source/target으로 방향이 있긴 한데, **역방향도 성립하는지** 표시 필요.
2. **부호 전파**: 금리↑ →(반비례)→ 채권↓ →(비례)→ ??? 이때 "반비례의 비례"는 "반비례"가 됨. 이 부호 계산 로직이 필요.

### ECO: 부호 전파는 조심해야 합니다

수학적으로:
- 비례 x 비례 = 비례 (+ x + = +)
- 비례 x 반비례 = 반비례 (+ x - = -)
- 반비례 x 반비례 = 비례 (- x - = +)

이건 1-2단계까지는 대체로 맞지만, 3단계 이상에서는 **현실과 괴리**가 생길 수 있어요. 그래서 3단계 이상은 "참고용"으로만 표시하고, 정확도가 떨어진다는 경고를 보여줘야 합니다.

### PM: 시나리오 시뮬레이션은 Phase 2가 맞나요? 꽤 중요한데...

"금리가 오르면?" 버튼 하나 누르면 전체 영향을 보여주는 건 매우 강력한 기능이에요. MVP에 넣을 수 없을까요?

### FE: 사실 연쇄 효과 기능의 확장이라서 가능합니다

연쇄 효과가 되면, "시작 노드 + 방향(↑/↓)" 입력 → BFS로 전파 → 각 노드의 영향 방향 표시. 이건 연쇄 효과 로직에 **방향(↑/↓) 파라미터**만 추가하면 됩니다.

다만 UI가 복잡해지니:
- MVP에서는 **노드 탭 → 팝업에 "이 지표가 오르면?" / "내리면?" 두 버튼**
- 버튼 클릭 시 연쇄 효과 + 각 노드에 ↑/↓ 아이콘 표시

### ECO: 이건 정말 킬 피처가 될 수 있어요

MVP에 넣읍시다. "금리가 오르면?" → 전체 관계도에 빨간/파란 화살표가 퍼지면서 각 지표가 어떻게 되는지 보여주는 거죠. **투자 초보가 가장 원하는 기능 1순위**입니다.

### DA: 기술적으로 가능합니다. 정리하면:

MVP 시뮬레이션 로직:
1. 사용자가 노드 선택 + 방향(↑/↓) 입력
2. BFS로 연결된 노드 탐색 (depth 제한: 3)
3. 각 링크의 type(비례/반비례)에 따라 부호 전파
4. 결과: 각 노드의 예상 방향(↑/↓/?)
5. 시각화: 노드에 화살표 아이콘, 선에 방향 애니메이션

## 합의된 MVP 기능 목록

### Must-Have (MVP)
1. **외부 데이터 로딩** — data.json에서 노드/링크 읽기
2. **Force-directed 레이아웃** — 자동 배치 + 드래그 조정
3. **노드 탭 → 하이라이트 + 상세 팝업** (개선)
4. **연쇄 효과 시각화** — 단계별 파급 효과 (depth 1-3)
5. **시나리오 시뮬레이션** — "이 지표가 오르면/내리면?" (연쇄 효과의 확장)
6. **카테고리 필터** — 특정 카테고리만 보기/숨기기
7. **검색** — 지표 이름으로 검색 → 해당 노드로 포커스
8. **관계 상세** — 링크 탭 시 관계 설명 팝업

### Should-Have (Phase 2)
- 경로 탐색 (A→B 경로 찾기)
- 사용자 메모
- 커스텀 노드/관계 추가 (관리 UI)
- 이벤트 기록
- economic-dashboard 데이터 연동
- 사용자 노드 위치 저장 (localStorage)

### Nice-to-Have (Phase 3)
- 톡방 캡처 → AI 분석 → 자동 추가
- 경제 용어 사전
- 스크린샷/시나리오 공유
- 학습 이력 추적
- 이벤트 알림

### 핵심 논쟁 기록
- **시나리오 시뮬레이션**: 원래 Phase 2였으나, 연쇄 효과의 자연스러운 확장이므로 MVP로 격상. ECO, PM 강력 지지, FE 기술적 가능 확인.
- **3단계 이상 연쇄**: ECO 경고 — 현실과 괴리 가능. "참고용" 경고 표시 합의.
- **부호 전파 로직**: 수학적으로 단순하지만 현실은 복잡. 조건부 관계는 시뮬레이션에서 "?" 표시.
