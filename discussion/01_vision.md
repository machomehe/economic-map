# 1. 프로젝트 비전과 핵심 가치

## 토론 참여자
- PM (기획자)
- FE (프론트엔드 개발자)
- ECO (경제 도메인 전문가)
- DA (데이터 아키텍트)

---

### PM: 문제 정의부터 시작하죠

투자 초보자가 경제 뉴스를 접할 때 가장 큰 문제는 **"금리가 올랐다"는 건 알겠는데, 그래서 내 투자에 뭐가 어떻게 되는지 모른다**는 겁니다. 경제 톡방에서 매일 인사이트를 듣지만 머릿속에서 체계화되지 않아요. 이 프로젝트의 핵심 가치는 **경제 지표 간의 인과관계를 시각적으로 보여줘서 "A가 변하면 B, C, D가 어떻게 되는지"를 직관적으로 이해시키는 것**이라고 봅니다.

궁극적 비전은: **"경제 톡방에서 배운 걸 캡처만 하면, AI가 분석해서 나만의 경제 관계도가 자동으로 완성되는 투자 매뉴얼"**

### ECO: 비전에 동의하지만, 위험한 함정이 있어요

방향은 맞는데 주의점이 있습니다. 경제 지표 관계는 **고정불변이 아닙니다**. 예를 들어:
- 2022년에는 금리 인상이 달러 강세로 직결됐지만
- 2024년 말에는 금리 인하 기대에도 달러가 강세였어요 (트럼프 정책 기대감)
- 금과 실질금리의 반비례 관계도 최근 몇 년간 깨졌습니다 (중앙은행 매입 때문)

그래서 단순히 "금리↑ → 달러↑"라고 보여주면 초보자가 **고정 공식으로 오해**할 수 있어요. **"일반적으로는 이렇지만, 지금 시장에서는 다른 요인이 작용 중"이라는 맥락**이 반드시 필요합니다.

### FE: 그렇다면 UI적으로 어떻게 표현할까요?

ECO의 포인트에 공감합니다. 현재 프로토타입은 관계를 "비례/반비례/조건부" 3가지로만 구분하는데, 이건 너무 단순해요. 제안:
1. **관계 강도** 표시 (선 굵기 or 색 진하기)
2. **시간 조건** 표시 ("단기에는 A, 중장기에는 B")
3. **현재 활성 상태** 표시 (지금 이 관계가 실제로 작동 중인지)

다만 모바일 화면에서 이걸 다 보여주면 정보 과부하가 됩니다. **기본 뷰는 단순하게, 탭하면 상세 정보**가 나오는 구조가 좋겠어요.

### DA: 데이터 관점에서 비전을 다시 정리하면

이 프로젝트는 사실 **두 가지 문제**를 풀어야 해요:

1. **정적 지식 베이스**: 경제 지표 간 일반적 관계 (교과서적)
2. **동적 컨텍스트**: 현재 시장 상황에서 어떤 관계가 활성화되어 있는지

지금 프로토타입은 1번만 하고 있어요. economic-dashboard에서 실시간 FRED 데이터를 이미 가져오고 있으니, 이걸 연동하면 2번도 가능합니다. 예: "현재 CPI가 3.2%로 상승 추세 → 인플레이션→금리 관계 활성화 중"

하지만 현실적으로 코딩 초보인 사용자가 유지보수할 수 있는 수준을 고려해야 합니다. 너무 복잡한 아키텍처는 안 됩니다.

### PM: 반론 — 처음부터 다 하려면 안 됩니다

DA 말이 맞지만, **MVP 관점에서 순서**를 정해야 합니다. 사용자의 일상 시나리오를 생각해보면:

1. 경제 톡방에서 "전쟁 나면 유가 올라" 같은 인사이트를 들음
2. 관계도를 열어서 "전쟁 → 유가 → 인플레 → 금리" 체인을 확인
3. "아, 그래서 내 성장주가 떨어질 수 있구나" 이해
4. 시간이 지나면서 자기만의 메모/관계가 추가됨

이 시나리오에서 가장 중요한 건 **2번 — 연쇄 관계 탐색**입니다. 지금 프로토타입에서 한 노드 터치하면 직접 연결만 보이는데, **2-3단계 연쇄 효과(cascading)**를 볼 수 있어야 해요.

### ECO: 연쇄 효과는 좋은데, 정확성 검증이 필요

"기준금리↑ → 달러↑ → 원화↓ → 외국인 매도↑ → 성장주↓" 이런 체인을 보여주면 직관적이긴 한데, 현실에서는 **각 단계마다 다른 변수가 개입**해요. 예를 들어 외국인이 원화 약세에도 매수하는 경우가 있고 (밸류에이션이 너무 싸졌을 때).

그래서 연쇄 효과를 보여줄 때 **"확신도" 또는 "이 경로가 현실에서 얼마나 자주 맞는지"**를 표시하면 좋겠어요. 물론 MVP에서는 정성적으로라도요.

### FE: 기술적으로 정리하면

현재 index.html 하나짜리 프로토타입에서 이 비전을 구현하려면:
- 데이터를 외부 JSON 파일로 분리해야 함 (지금은 하드코딩)
- 관계 탐색을 위한 그래프 알고리즘 필요 (BFS/DFS for cascading)
- 모바일에서 D3.js나 force-directed graph가 성능이 괜찮을지 검증 필요

다만 사용자가 코딩 초보이므로, **빌드 도구 없이 정적 파일만으로** 동작하는 구조를 유지하는 게 중요합니다.

### DA: 동의합니다. CDN + 정적 JSON 구조로 가야 해요

GitHub Pages로 배포하고 있으니:
- `data/nodes.json`, `data/links.json` 으로 데이터 분리
- JS 라이브러리는 CDN으로
- 빌드 과정 없이 HTML + JS + JSON만으로 구성

사용자가 나중에 데이터만 수정해서 관계도를 업데이트할 수 있는 구조가 이상적입니다.

---

## 합의된 비전

> **"경제 지표 간의 인과관계를 직관적으로 시각화하여, 투자 초보자가 하나의 경제 이벤트가 자신의 투자에 미치는 연쇄 영향을 즉시 파악할 수 있게 하는 인터랙티브 관계도"**

### 핵심 가치 3가지
1. **직관성**: 터치 한 번으로 연쇄 관계를 시각적으로 파악
2. **정확성**: 단순 공식이 아닌, 조건과 맥락이 포함된 관계 표현
3. **성장성**: 사용자가 학습하면서 관계도가 함께 커가는 구조

### 차별점
- 기존 교과서적 관계도 + 실시간 시장 맥락
- 톡방 캡처 → AI 분석 → 자동 추가 (장기 비전)
- economic-dashboard의 실시간 데이터와 연동 (중기 비전)

### 제약 조건
- 코딩 초보가 유지보수 가능한 구조 (빌드 도구 없음)
- 모바일(폰) 우선
- GitHub Pages 정적 배포
- 점진적 고도화 (한 번에 다 하지 않음)
